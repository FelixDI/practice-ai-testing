description: 按四维覆盖 + 优先级检查模块测试用例是否有遗漏
---

你是一位资深测试工程师。请严格按以下要求，对 **$ARGUMENTS** 模块的测试用例进行四维覆盖审计。

## 1. 遵守规范

- 首先，读取项目根目录下的 CLAUDE.md，了解四维覆盖模型、优先级分层策略和用例文档格式。
- 其次，读取全局技能文件 ~/.claude/skills/python-automation-test-standard/SKILL.md，了解三类必覆盖场景。
- 接口/页面信息：**API 测试**来源于 `docs/practice_software_testing_api.json` 中 $ARGUMENTS 模块的 paths 和 schemas；**UI 测试**参考 CLAUDE.md 页面清单和 Playwright 实测导航。

## 2. 审计流程

1. 读取 `docs/test-cases/$ARGUMENTS.md`，统计已有用例及覆盖的优先级（P0/P1/P2/P3）。
2. 读取 API 文档中该模块的全部端点，确认每个端点至少有 2~3 条 P0 用例。
3. 逐维度检查遗漏：
   - **① 核心链路**：每个端点的 Happy Path（200/201/204）是否都有 P0 覆盖。
   - **② 异常路径**：401/404/409/422 是否按端点逐一覆盖（P1）。
   - **③ 边界条件**：Schema 字段的 maxLength/minLength/format/range/enum 是否逐字段遍历（P2）。
   - **④ 权限/状态组合**：越权、Token 过期、注入攻击是否覆盖（P1+P3）。
4. 输出审计报告，列出已覆盖项和遗漏项。

## 3. 输出格式

```
## $ARGUMENTS 模块审计报告

### 端点覆盖率
| 端点 | P0 | P1 | P2 | P3 | 状态 |
|------|:--:|:--:|:--:|:--:|------|

### 四维覆盖
| 维度 | 已覆盖 | 遗漏 |
|------|------|------|
| ① 核心链路 | ... | ... |
| ② 异常路径 | ... | ... |
| ③ 边界条件 | ... | ... |
| ④ 权限组合 | ... | ... |

### 结论
- 总用例数：N
- 遗漏数：N
- 建议补充的用例列表
```

## 4. 处理遗漏

- 如果遗漏 ≤ 5 条：直接补充到 `docs/test-cases/$ARGUMENTS.md`，然后为新增用例编写测试脚本，用 MCP 工具 `run_pytest($ARGUMENTS)` 验证修复直至通过。
- 如果遗漏 > 5 条：列出遗漏清单，询问是否逐项补充。

## 5. 验证（强制执行）

- **单文件**：Hook 已在每次 Write/Edit 后自动跑过被修改文件，确保逐文件即时反馈。
- **模块级回归**：所有新增测试脚本编写完成后，用 MCP 工具 `run_pytest($ARGUMENTS)` 做模块级回归确认，分析失败原因并修复，直到全部用例通过。
